# AgentWorld 系统概要设计（全量）

## 1. 目标与边界

AgentWorld 是团队级 Agent 平台，目标是把 Agent 从“个人调用”升级为“团队可治理、可审计、可扩展的任务执行系统”。

边界：
- 核心仓库负责主干编排、治理、观测、任务执行。
- 外部能力（IM、邮件、代码仓、Provider）以插件化方式接入，避免侵入主干。

## 2. 九层架构

1. Provider 执行层
2. Agent 定义层
3. 工具/Skill 管理层
4. 多 Agent 编排层
5. Agent 团队任务执行层
6. 业务团队管理层
7. 任务执行展示层
8. 环境层
9. 记忆层

设计原则：
- 一层一职责：每层只承担单一主责。
- 可观测优先：任务过程可回放、可解释、可追责。
- 插件优先：扩展能力优先走插件挂载，而不是改主干代码。

## 3. 前后端排布

- 前端：按九层 + 任务中心组织导航。
- 后端：
  - `*-core.ts` 为主干领域逻辑。
  - `*-adapter.ts` 为外部系统接入边界。
  - `api/**` 提供能力编排和读写接口。

## 4. 可靠性与可扩展性

- 幂等：Quest 提交和执行节点具备幂等语义。
- 重试：失败节点可独立重试，避免整任务重跑。
- 干预：人工门禁与手工恢复能力。
- 可扩展：Provider、工具、通知、环境都支持独立扩展。

## 5. 两个标准案例

### 5.1 神盾计划（MR 检视）
- 触发：Webhook。
- 输入：MR diff / 变更上下文。
- 执行：多 Agent 分层审查。
- 输出：MR 评论 + 审查证据链 + 记忆沉淀。

### 5.2 安全检视系统
- 触发：定时。
- 输入：全量代码仓。
- 执行：安全规则扫描 + 研判。
- 输出：风险报告 + 邮件通知 + 长期风险画像。
