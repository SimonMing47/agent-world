# AgentWorld 控制台视觉设计规格

## 1. 目标

把 AgentWorld 控制台统一为一套可长期扩展的团队级 B 端产品视觉系统。整体目标不是做展示站，而是做一套具有以下特征的工作台：

- 简约：减少解释性语言和无意义装饰
- 强调色克制：只把强调色留给主动作、焦点状态和关键指标
- 设计感：通过比例、留白、层次和控件纪律体现完成度
- 专业感：所有页面优先服务任务治理、智能体治理、团队治理和基础配置

## 2. 风格原则

### 2.1 产品气质

- 欢迎页负责品牌进入感
- 登录页负责进入系统感
- 控制台负责秩序感和效率感
- 内部页面不再承担“解释系统是什么”的职责，只承担“让用户开始工作”的职责

### 2.2 视觉主线

- 基底为浅冷灰画布
- 卡片为纯白工作面
- 文本以高对比黑灰体系组织层级
- 只保留一支深蓝作为系统强调色
- 通过留白、对齐和边界节制，而不是大面积高饱和装饰，建立高级感

## 3. 色彩系统

### 3.1 中性色

- 画布背景：`#F5F5F7`
- 主表面：`#FFFFFF`
- 次表面：`#F8F9FB`
- 弱表面：`#F0F1F4`
- 主文字：`#111111`
- 次文字：`#69707D`
- 弱文字：`#8F96A3`
- 分隔线：`rgba(15, 23, 42, 0.055)`

### 3.2 强调色

- 强调色：`#2457FF`
- 强强调色：`#1947DF`
- 弱强调背景：`#EDF2FF`

### 3.3 使用纪律

- 蓝色仅用于主按钮、当前激活、链接焦点、关键状态
- 普通标签默认中性灰
- 指标数字默认使用主文字，不使用彩色
- 危险色仅用于删除、失败、不可逆操作

## 4. 排版系统

### 4.1 页面层级

- 页面标题：`28-32px / semibold / tight`
- 模块标题：`18px / semibold`
- 行内标题：`14-16px / medium`
- 辅助文案：`14px / line-height 24`
- 小标签与眉题：`11px / uppercase / tracking 0.12em`

### 4.2 文案规则

- 每个页面只保留一句必要描述
- 不在首屏写大段“系统说明”
- 空状态文案控制在两句以内
- 按钮文案尽量使用动词，不使用解释句

## 5. 结构系统

### 5.1 页面结构

所有内部页面遵循统一结构：

1. 页面头部
2. 核心指标带
3. 主工作区
4. 次级表格或详情区

### 5.2 页面头部

- 左侧：眉题、标题、简短描述
- 右侧：状态 badge、主操作、次操作
- 不再出现超大口号式 Hero

### 5.3 主工作区

- 优先使用表格、列表、工作面板
- 避免用大卡片承载空内容
- 操作入口应靠近内容本体

## 6. 控件规范

### 6.1 按钮

- 主按钮：深蓝底白字，仅一屏一个主动作
- 次按钮：白底细边框
- Ghost：仅用于行内轻操作
- 危险按钮：只在删除确认前后出现

### 6.2 Badge

- 只作为状态或轻标签使用
- 不承担主层级信息
- 默认中性灰，运行态/选中态可使用强调色

### 6.3 卡片与面板

- 标准圆角：`22px`
- 内间距：`20-24px`
- 阴影极轻，仅做层级
- 面板头部保持白色，不做重色块分区

### 6.4 表格

- 业务页优先用表格展示已配置数据
- 表头统一为小号大写眉题风格
- 纵向不画分割线
- 行 hover 只做极轻底色变化
- 行操作放在最右侧，默认弱化

## 7. 空状态规范

- 空状态统一为轻量虚线容器
- 不使用大面积空卡片撑布局
- 空状态只包含：图标、标题、短说明、一个动作

## 8. 导航壳层规范

### 8.1 左侧边栏

- 固定分为：总览、智能体治理、团队治理、基础配置
- 选中态使用极轻强调背景和细竖线
- 分组标题弱化
- 菜单文字比当前略收紧，减少“后台清单感”

### 8.2 顶部栏

- 仅保留当前页面名称和右上角用户入口
- 不加入多余说明
- 背景轻毛玻璃，保持稳定秩序感

## 9. 页面落地策略

### 9.1 总览页

- 去 Hero 化
- 首屏直接展示任务摘要、优先任务、当前状态、最近会话、团队分布、任务蓝图
- 用表格和紧凑列表代替大面积概念展示

### 9.2 模型对话页

- 去说明化
- 单 Agent 与 Agent 团队入口直接工作台化
- 最近会话和全部会话表格化

### 9.3 团队页

- 表格优先，树形结构作为补充
- 删除操作后置
- 团队情况直接在详情表和指标中体现

### 9.4 知识库页

- AgentWorld 知识引擎 状态降级为 badge
- 知识空间和知识条目优先表格化
- 使用关系和消费者关系直接表格可见

## 10. 交互要求

- 页面切换优先体感流畅，使用预取和轻量 loading
- 弹窗统一为系统面板风格
- 滚动时布局稳定，不因为内容高度变化跳动
- 会话、运行、任务等高频页面优先保证操作反馈即时

## 11. 适用范围

本规格适用于：

- 全局壳层
- 欢迎页与登录页
- 总览、模型对话、团队、知识库等高频控制台页面
- 未来所有新增治理页面与配置页面
